
**Database schema (PostgreSQL 16 via Flyway):**

| Table | Purpose |
|---|---|
| `users` | Accounts — username, email, bcrypt hash, display name, avatar |
| `nodes` | One row per active session — model list (TEXT[]), vram_gb, last_seen |
| `peer_groups` | Named groups with an owner |
| `group_members` | Many-to-many users↔groups with role + joined_at |
| `conversations` | Metadata per conversation (title, model used) |
| `messages` | Individual messages with role, content, from_peer flag |

**REST API surface:**

| Method | Endpoint | Auth | Description |
|---|---|---|---|
| POST | `/api/auth/register` | No | Create account |
| POST | `/api/auth/login` | No | Login → JWT |
| GET | `/api/users/me` | JWT | Get profile |
| PUT | `/api/users/me` | JWT | Update display name / avatar |
| POST | `/api/nodes/heartbeat` | JWT | Register/refresh node as online |
| GET | `/api/nodes/{groupId}` | JWT | List online nodes in a group (2-min window) |
| POST | `/api/groups` | JWT | Create group |
| POST | `/api/groups/{id}/invite` | JWT | Invite user by username (owner only) |
| GET | `/api/groups/mine` | JWT | List groups user belongs to |
| GET | `/api/conversations` | JWT | List conversations |
| POST | `/api/conversations` | JWT | Create conversation |
| POST | `/api/conversations/{id}/messages` | JWT | Save message |

**Design decisions:**
- Table named `peer_groups` instead of `groups` — avoids reserved-keyword edge cases with Hibernate/JPA
- JWT key derived via SHA-256 hash of the secret string so any secret length works with HS256
- Node online window is 2 minutes (configurable in `NodeController`)
- Owner is automatically added as `admin` member when creating a group

### 2. OpenClaw FastAPI Backend (`openclaw/`)

Python 3.12 / FastAPI local AI backend. Wraps Ollama's streaming API and persists conversations to local SQLite.

**Files created:**
```
openclaw/
├── Dockerfile          (python:3.12-slim, hot-reload mount in docker-compose)
├── requirements.txt    (fastapi, uvicorn, httpx, aiosqlite, sse-starlette, PyMuPDF)
└── app/
    ├── __init__.py
    ├── server.py       (FastAPI app, CORS middleware, lifespan → init_db)
    ├── database.py     (SQLite init, 4 tables: sessions, messages, settings, agent_log)
    └── routers/
        ├── __init__.py
        ├── chat.py     (session CRUD + streaming SSE chat endpoint)
        ├── models.py   (GET /api/models → Ollama /api/tags)
        └── upload.py   (POST /api/upload → PDF/TXT/CSV text extraction)
```

**Local SQLite schema:**

| Table | Purpose |
|---|---|
| `sessions` | Conversation sessions — model, system_prompt, temperature |
| `messages` | Full message content (plaintext, never sent to cloud) |
| `settings` | Key-value store for user preferences |
| `agent_log` | Audit trail for future agent mode actions |

**Key endpoint — streaming chat:**
- `POST /api/sessions/{id}/chat` — builds history from SQLite, forwards to Ollama with `"stream": true`, proxies SSE chunks to the browser, saves the completed assistant response to SQLite after the stream ends.
- Graceful error if Ollama is not running (503 with helpful message)

**File upload:**
- `.txt` / `.md` → decoded as UTF-8
- `.pdf` → text extracted via PyMuPDF (fitz)
- `.csv` → raw text, capped at 50k chars

### 4. Docker Compose (`docker-compose.yml`)

4-service stack, all wired together.

| Service | Port | Notes |
|---|---|---|
| `db` | 5432 | PostgreSQL 16 with healthcheck |
| `backend` | 8080 | Spring Boot — waits for db healthy |
| `openclaw` | 8000 | FastAPI — connects to Ollama at `host.docker.internal:11434` |
| `frontend` | 3000 | nginx serving React build, proxying /api → openclaw |

`openclaw` mounts `./openclaw/app` as a volume so Python changes hot-reload without a rebuild during development.

## How to Run

**Prerequisites:**
1. Install Ollama from [ollama.com](https://ollama.com) (native macOS ARM64 build)
2. Install Docker Desktop

**Start Ollama (native, uses Metal GPU):**
```bash
ollama serve
ollama pull llama3.2:3b
```

**Start everything else:**
```bash
cd /Users/isaacamar/Documents/meshmind-main
docker compose up --build
```

| URL | What |
|---|---|
| http://localhost:3000 | Chat UI |
| http://localhost:8000 | OpenClaw API |
| http://localhost:8080 | Spring Boot API (Postman demo) |
| localhost:5432 | PostgreSQL |

**Quick Postman demo (Spring Boot):**
```bash
# Register
curl -X POST http://localhost:8080/api/auth/register \
  -H "Content-Type: application/json" \
  -d '{"username":"alice","email":"alice@test.com","password":"password123"}'

# Login
curl -X POST http://localhost:8080/api/auth/login \
  -H "Content-Type: application/json" \
  -d '{"username":"alice","password":"password123"}'

# Heartbeat (use token from login)
curl -X POST http://localhost:8080/api/nodes/heartbeat \
  -H "Authorization: Bearer <token>" \
  -H "Content-Type: application/json" \
  -d '{"modelList":["llama3.2:3b"],"vramGb":24.0}'

# Create group
curl -X POST http://localhost:8080/api/groups \
  -H "Authorization: Bearer <token>" \
  -H "Content-Type: application/json" \
  -d '{"name":"ECE366 Team","description":"MeshMind dev group"}'
```

## Runtime Notes (from actual test run)

### Spring Boot — confirmed working
- Container started in **2.014 seconds**
- Flyway ran `V1__init.sql` cleanly on fresh PostgreSQL: `Successfully applied 1 migration to schema "public", now at version v1`
- Hibernate validated all 6 JPA entities against the schema with no errors
- JWT filter registered, security filter chain active
- Tested endpoints live:
  - `POST /api/auth/register` → 201, user created with UUID
  - `POST /api/auth/login` → 200, JWT token returned
  - All subsequent calls require `Authorization: Bearer <token>`

### Ollama — confirmed working on M4 Pro
- Installed via `brew install ollama` (version 0.18.2)
- Metal GPU detected: **Apple M4 Pro, 16.0 GiB available VRAM**
- **Important:** Ollama must start with `OLLAMA_HOST=0.0.0.0 ollama serve` so Docker containers can reach it via `host.docker.internal:11434`. Default `127.0.0.1` binding is unreachable from inside Docker on macOS.
- Recommended start command (enables flash attention + quantized KV cache for M4):
  ```bash
  OLLAMA_HOST=0.0.0.0 OLLAMA_FLASH_ATTENTION=1 OLLAMA_KV_CACHE_TYPE=q8_0 ollama serve
  ```

### Model recommendation for demo
- `llama3.2:3b` — 2GB, very fast on M4, good for demos
- `llama3.1:8b` — 5GB, better quality, still fast on M4 Pro with 16GB VRAM
- Pull with: `ollama pull llama3.2:3b`

### Full local stack — how to run
```bash
# Terminal 1: Ollama (native, Metal GPU)
OLLAMA_HOST=0.0.0.0 ollama serve

# Terminal 2: pull model (first time only)
ollama pull llama3.2:3b

# Terminal 3: Docker stack
cd /path/to/meshmind-main
docker compose up --build

# Browser
open http://localhost:3000
```
